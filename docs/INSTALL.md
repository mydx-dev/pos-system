# インストールガイド

本システムは、Google スプレッドシートをデータベースとして利用し、Google Apps Script の Web アプリとして動作します。

---

## 1. 事前準備

利用開始には、以下が必要です。

- Google アカウント
- Google スプレッドシートを作成・編集できる権限
- Google Apps Script プロジェクトを作成・編集できる権限
- 配布された本番ビルド成果物一式

管理者が導入する場合は、あわせて以下も必要です。

- 利用者へ共有する Google グループまたは Google アカウント一覧
- 初期管理者として登録するユーザー情報

---

## 2. 導入方法

本システムは、以下の 2 つの方法で導入できます。

| 方法                         | 概要                                                        | 推奨ケース                                                           |
| ---------------------------- | ----------------------------------------------------------- | -------------------------------------------------------------------- |
| スプレッドシートをコピーする | 既存のテンプレートスプレッドシートをコピーして利用する      | 最も簡単に導入したい場合                                             |
| 成果物をコピーして設定する   | 配布されたビルド成果物を Apps Script プロジェクトへ配置する | 独自のスプレッドシート構成・Apps Script プロジェクトで利用したい場合 |

通常は **スプレッドシートをコピーする方法** を推奨します。

---

### テンプレートをコピーする場合

管理者から共有された[テンプレートスプレッドシート]()を開きます。

Google スプレッドシートのメニューから、以下を選択してください。

```text
ファイル → コピーを作成
```

コピー先の名前と保存先を指定し、スプレッドシートをコピーします。

> コピー後のスプレッドシートが、本システムで利用するデータベースになります。

---

### 成果物をコピーして導入する場合

この方法は、配布されたビルド成果物を既存または新規の Apps Script プロジェクトへ配置して導入する方法です。

#### 配布物を確認する

配布された成果物には、通常、以下が含まれます。

```text
dist/
├── appsscript.json
├── index.html
├── *.js
├── *.html
└── assets/
```

ファイル構成はリリースバージョンによって異なる場合があります。

---

#### Apps Script プロジェクトを作成する

作成したスプレッドシートから、以下を選択します。

```text
拡張機能 → Apps Script
```

Apps Script プロジェクトを作成します。

---

#### ビルド成果物を配置する

配布された成果物を Apps Script プロジェクトへ配置します。

`clasp` を利用する場合は、成果物ディレクトリを指定して push します。

```bash
clasp push
```

手動で配置する場合は、成果物内の JavaScript・HTML・設定ファイルを Apps Script エディタへコピーします。

> 手動配置はファイル数が多くなるため、可能な限り `clasp` を利用してください。

---

## 3. 公開設定

### Apps Script プロジェクトを開く

コピーしたスプレッドシートを開き、以下のメニューを選択します。

```text
拡張機能 → Apps Script
```

Apps Script エディタが開きます。

---

### Web アプリとしてデプロイする

Apps Script エディタ右上の **デプロイ** を選択し、以下の手順で Web アプリを公開します。

```text
デプロイ → 新しいデプロイ
```

デプロイタイプとして **ウェブアプリ** を選択します。

設定例：

| 項目                   | 設定                           |
| ---------------------- | ------------------------------ |
| 説明                   | 初回リリース、または任意の説明 |
| 次のユーザーとして実行 | 自分として実行                 |
| アクセスできるユーザー | 全員                           |

設定後、**デプロイ** を選択します。

初回デプロイ時には、Google アカウントによる認可が必要です。画面の案内に従って必要な権限を許可してください。

---

### Web アプリ URL を確認する

デプロイ完了後に表示される Web アプリ URL を控えてください。

```text
https://script.google.com/macros/s/{DEPLOYMENT_ID}/exec
```

この URL が利用者向けのアクセス URL です。

---

### システムセットアップ

初回利用前に、システムの初期設定をします。

1. Web アプリにアクセスします
2. 画面に従い、初期設定を実行します

> [!CAUTION] 必ずスプレッドシートのオーナーGoogleアカウントでブラウザにログインした状態で実行してください。

---

## 6. 動作確認

導入後、以下を確認してください。

### 6.1 ログイン確認

- Web アプリ URL にアクセスできる
- 対象の Google アカウントでログインできる
- エラー画面が表示されない

### 6.2 データ確認

- 案件一覧を表示できる
- 見積一覧を表示できる
- ユーザー一覧を表示できる

---

## 7. アップデート方法

新しいバージョンが配布された場合は、以下の手順で更新します。

1. 最新のリリース成果物を取得する
2. Apps Script プロジェクトへ成果物を反映する
3. 新しいバージョンとしてデプロイする
4. Web アプリ URL が変わっていないことを確認する
5. 必要に応じて利用者へ更新内容を案内する

既存の Web アプリ URL を維持したい場合は、既存デプロイを更新してください。

```text
デプロイ → デプロイを管理 → 編集 → 新しいバージョン
```

---

## 8. トラブルシューティング

### Web アプリにアクセスできない

以下を確認してください。

- Web アプリのアクセス権設定
- Google アカウントが許可対象に含まれているか
- 組織の Google Workspace ポリシー
- Web アプリ URL が正しいか

---

### 「権限がありません」と表示される

Google アカウントでログインできていても、システム上の権限が不足している可能性があります。

管理者に以下を確認してください。

- ユーザーがシステムに登録されているか
- 必要なロールまたは権限が付与されているか
- 対象の操作が許可されているか

---

### データが表示されない

以下を確認してください。

- 必要なシートが存在するか
- シート名が変更されていないか
- 列名・列順が変更されていないか
- Apps Script の実行ログにエラーが出ていないか

Apps Script エディタから、以下を確認できます。

```text
実行数 → 実行履歴
```

---

### デプロイ後に変更が反映されない

Apps Script は、コードを更新しただけでは Web アプリに反映されません。

以下を実施してください。

```text
デプロイ → デプロイを管理 → 編集 → 新しいバージョン
```

更新後、ブラウザのキャッシュが残っている場合は、スーパーリロードまたはシークレットウィンドウで確認してください。

---

## 9. 問い合わせ先

導入・権限設定・不具合に関する問い合わせは、システム管理者または導入担当者へ連絡してください。
